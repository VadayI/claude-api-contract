## 3. Bugs Found and Fixed

### Bug #1 — HIGH: `bootstrap.md` Mode A references non-existent templates (FIXED in this PR)

**File:** `.claude/commands/bootstrap.md`, Mode A Step 2.

**Problem:** Step 2 claimed to scaffold `spec/main.tsp`, `spec/models/`, `spec/auth.tsp`, a sample
resource, and `examples/` **"from `templates/`"**. In reality `templates/` holds only
`PROJECT.md`, `output-language.md`, and `plan.md`. The spec/examples templates were removed in
commit `11d7ba1` and not relocated. On a fresh clone, Step 2 was unexecutable as written.

Also incorrectly listed as needing to be "copied" from templates: `.spectral.yaml`,
`docs/api/INDEX.md`, `.github/workflows/contract-ci.yml`, `CHANGELOG.md` — these already ship in the
cloned scaffold.

**Fix:** Rewrote Step 2 to:
- Explicitly list which files already exist in the scaffold (no action needed).
- Delegate `spec/` and `examples/auth/` authoring to `tsp-author` via the pipeline, guided by
  `auth-contract.md` and `api-envelope.md`. The first domain resource is still authored later via
  the full `ba → api-architect → tsp-author` pipeline.

This is consistent with the contract-first / no-stubs philosophy.

### Bug #2 — LOW: Rule-scoping documentation drift in `CLAUDE.md` (FIXED in this PR)

**File:** `CLAUDE.md`, "Claude-specific behavior" → rule-scoping paragraph.

**Problem 1:** `typespec-style.md` was listed as loaded by `(tsp-author, contract-reviewer)`, but
`contract-reviewer.md` did not reference it. The rule is genuinely relevant to contract review
(TypeSpec authoring conventions, snake_case, `@doc`, named models, `operationId` stability) — the
agent file just lacked the explicit link.

**Fix:** Added a **TypeSpec style** bullet to `contract-reviewer.md` checklist with the
`@.claude/rules/typespec-style.md` reference.

**Problem 2:** `examples-validation.md` was listed as `(mock-validator, docs-writer)`. Actual
consumers: `mock-validator`, `tsp-author`, `fix-ci`, `validate-contract`. `docs-writer` was listed
but did not reference the rule; the other three were omitted from the docs.

**Fix:**
- Added an explicit rule reference to `docs-writer.md` (verify examples before referencing in docs).
- Updated `CLAUDE.md` rule-scoping paragraph to list all five actual consumers:
  `(mock-validator, docs-writer, tsp-author, fix-ci, validate-contract)`.

## 5. Temporary Files — Two Classes

After a fresh clone + `npm ci` + `session-start.sh`, the following files are created. They can be
safely removed at any time.

### Class A — Build / Session Artifacts (always safe to delete; regenerated automatically)

| Path | Created by | Regenerated by |
|---|---|---|
| `node_modules/` | `npm ci` / `npm install` | `npm ci` |
| `tsp-output/` | `npm run api:compile` | `npm run api:compile` |
| `.tsp/` | TypeSpec compiler cache | `npm run api:compile` |
| `.claude/memory/env-detect.json` | `session-start.sh` → `detect-env.mjs` | Session start (auto) |
| `.claude/memory/command-log.jsonl` | First `/`-command run | Auto-appended each command |
| `.env` | `session-start.sh` seed from `.env.example` | `session-start.sh` if `.env` is absent |

**Cleanup:**
```bash
rm -rf node_modules tsp-output .tsp .env
rm -f .claude/memory/env-detect.json .claude/memory/command-log.jsonl
```

### Class B — Template-Development Leftovers (only relevant on the template's own working copy)

These do **not exist** on a fresh clone. They exist in the template's local working copy only
because `spec/`, `examples/`, and ADRs 0002–0004 were removed from git tracking in `11d7ba1` but not
deleted from disk. Delete them only if you want this copy to match the state of a fresh clone.

| Path | Notes |
|---|---|
| `LOCAL/` | Developer notes, requirements — gitignored, local-only |
| `spec/` | Reference articles+auth implementation, untracked since `11d7ba1` |
| `examples/` | Reference examples, untracked since `11d7ba1` |
| `openapi.yml` | Build output of the reference `spec/`, untracked since `11d7ba1` |
| `docs/decisions/0002-contract-first-source-of-truth.md` | Project-level ADR, untracked |
| `docs/decisions/0003-auth-bearer-jwt-refresh-in-body.md` | Project-level ADR, untracked |
| `docs/decisions/0004-version-prefix-auth-paths.md` | Project-level ADR, untracked |
| `.claude/memory/endpoints.json` | Pipeline output; untracked locally (absent on fresh clone) |
| `.claude/settings.local.json` | Machine-local settings, gitignored |

**Cleanup** (brings local copy to fresh-clone state — **irreversible**):
```bash
rm -rf LOCAL spec examples tsp-output
rm -f openapi.yml .env .claude/settings.local.json .claude/memory/endpoints.json
rm -f docs/decisions/0002-contract-first-source-of-truth.md
rm -f docs/decisions/0003-auth-bearer-jwt-refresh-in-body.md
rm -f docs/decisions/0004-version-prefix-auth-paths.md
```
